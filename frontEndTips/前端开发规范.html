    # 前端文件组织结构

    前端项目根目录下各文件夹/文件内容如下：

    |-- App.vue
|-- api --与后端交互API
|-- assets --前端静态资源
|-- components --通用组件
|-- config --全局配置

|-- directive --自定义指令
|-- doc --storybook文档
|-- error.js

|-- mixins --混入

|-- permission.js

|-- store --vuex相关配置
|-- stories --storybook相关配置

|-- util --工具文件
`-- views --开发vue代码放置位置

    # 内置实体组织结构

    # 只有按钮以及右键菜单不同的内置实体

    在实体中新增按钮配置，以及右键菜单配置，通过在实体中配置为特殊的内置实体添加不同的按钮、按钮回调事件、不同的右键菜单、菜单回调事件来对不同类型的内置实体进行处理操作，该部分需要修改代码结构，如确认采取该方案，则给出详细的设计方案。

    # 差别较大的内置实体

    对于需要大量开发量开发的内置实体如树实体等，将实体放置在buildEntity，通过设置路由条件展示对应实体页面，所有内置实体均放置在buildEntity文件夹下，实体的目录与剧本实体目录相同。

    Todo: 在buildInEntity下提供一个内置实体demo，通过配置实体路由，以及复制该实体下的代码，即可在它的基础上修改，开发内置实体

    # 其他系统实体

    如果其他实体上具有特殊功能，需要在views下文件夹下标识实体，需要重写的代码统一在该文件夹内部进行处理，按照需要导入实体所需功能，通过实体标识配置路由。

    # 命名规范

    1. 列表页面以 index.vue命名，详情页面以 detail.vue命名

    2. 引用类型变量应标识出数据类型：如 xxxList，xxxObject , xxxDict等，变量采用驼峰方式命名

    3. 函数名称以翻译软件翻译的英文命名，处理函数以handle开头，如(handleClick()等)

    4. 引用组件以html规范的组件标识，多个词语以 - 分隔，如

    5. 基础组件以base开头命名，如

    # 注释规范

    1. 定义的全局变量应给出注释，说明变量含义

    2. 定义的函数应当给出注释，注释内容包括：函数功能、函数变量、函数返回值等

    3. 复杂的函数应当在注释中给出设计说明，说明函数的执行逻辑

    4. 变量注释标识在变量之后，函数注释标识在函数定义上方

    5. 在vue页面上使用插件增加注释，标识页面说明、修改人员、修改日期等信息

    # 代码结构规范

    1. 公共函数应当提取形成公共js，以减少代码冗余

    2. 不在页面中使用行内样式
