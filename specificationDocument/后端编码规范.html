    后端编码规范

    # 后端编码规范

    # 版本控制规范

    1. 为产品添加新功能时，应在git内创建新分支，测试稳定后合并至主干

    # 基本要求

    1. 【强制】遵守DRY(Don't Repeat Yourself)原则。

    2. 【强制】Controller类里不能出现业务逻辑，只能直接调用Service类；业务逻辑需要放在Service层

    3. 【强制】Interface接口定义类只定义方法，方法的实现要在实现类里完成，Interface接口以xxxService结尾，接口实现类以xxxServiceImpl结尾。

    4. 【强制】 代码应当具备高内聚，低耦合的特性

    5. 【推荐】 每一个方法，代码量不超过60行，每一个编译模块，代码量不超过600行

    6. 【推荐】 计算尽量放在循环外执行，降低代码计算时间复杂度

    # 命名风格

    1. 【强制】类名使用 UpperCamelCase 风格

      ForceCode / UserDO / HtmlDTO / XmlService / TcpUdpDeal / TaPromotion

    2. 【强制】 方法名、参数名、成员变量、局部变量都统一使用 lowerCamelCase 风格（除第一个单词之外，其他单词首字母大写）

      localValue / getHttpMessage() / inputUserId

    3. 常量命名全部大写，单词间用下划线隔开，力求语义表达完整清楚，不要嫌名字长。

      MAX_STOCK_COUNT / CACHE_EXPIRED_TIME常量定义

    # 常量定义

    1. 【强制】不允许任何魔法值（即未经预先定义的常量）直接出现在代码中。

    # 变量定义

    1. 【推荐】尽量使用局部变量，减少全局变量的使用

    2. 【强制】在使用局部变量的过程，按就近原则处理。不建议定义一个局部变量，然后在很远的地方才使用

    3. 【强制】局部变量与全局变量在使用时，应当正确初始化

    4. 【强制】变量的静态/非静态属性应当保证正确

    # 方法定义

    1. 【强制】每个方法的参数值在使用之前均应当检查

      public R addComment(@RequestBody DisscussVO vo) {
    if(StringUtils.isBlank(vo.getDiscussEntity())){throw new CustomException("实体id不能为空！");}
    if(StringUtils.isBlank(vo.getDiscussObject())){throw new CustomException("实例id不能为空！");}
    if(StringUtils.isBlank(vo.getDiscussFlag())){throw new CustomException("评论标识符不能为空！");}
    ...
    return R.ok(iDynamicDiscussService.addComment(vo));

    2. 【强制】静态方法、非静态方法应当定义清楚

    # 代码格式

    1. 【强制】if、for、do、while等语句的执行语句部分无论多少都要加括号"{}"。

      for (TTreeObjectDTO process : ans) {
    if(process.getChildren()==null){
        process.setLeafNodeFlag(true);
        if(process.getPData1()!=null && process.getPData1()!=0){
            mapLeaf_Inverse.put(process.getPData1(), process); //节点的diagramId与TTreeObjectDTO的关系
        }
    }

    2. 【强制】在使用局部变量的过程，按就近原则处理。不建议定义一个局部变量，然后在很远的地方才使用

      if (LeftTreeTypeEnum.ENTITY_OBJECT_LEFT_TREE.getType().equals(leftTreeType)) {

    } else {
        ...
    }
}

    # OOP 规约

    1. 【强制】所有的覆写方法，必须加@Override 注解

      @Override
public List<Long> getStartEndObjectIds(List<Long> objectIds){
    List<Long> ans=new ArrayList<>();
    ans.addAll(this.getBaseMapper().getStartObjectIds(objectIds));
    ans.addAll(this.getBaseMapper().getEndObjectIds(objectIds));
    return ans;
}

    2. 【强制】对外暴露的接口签名，原则上不允许修改方法签名，避免对接口调用方产生影响。接口过时必须加@Deprecated 注解，并清晰地说明采用的新接口或者新服务是什么

      /**
 * Get paged users.
 * @param pageNo number index of page
 * @param pageSize size of page
 * @return A collection of users, empty set if no user is found
 * @since 1.2.0
 * @接口说明，该接口已弃用，新接口详见: "ResetResultNew()"
 */
@PutMapping("/password")
@Deprecated
public RestResult<String> updatePassword(@RequestParam(value = "oldPassword") String oldPassword,
        @RequestParam(value = "newPassword") String newPassword) {
    ...
}

    3. 【强制】所有整型包装类对象之间值的比较，全部使用 equals 方法比较

      public static void checkRequiredAndLengthNum(Object obj, int len, String message) {
    if (null == obj || "".equals(obj.toString())) {
        throw new CustomException(message+"字段必填",300);
    }
    if(obj.toString().length()>len){
        throw new CustomException(message+"字段超过最大长度"+len,300);
    }
}

    4. 【推荐】 把子类中的公用方法放到父类中处理

    # 控制语句

    1. 【强制】每个switch语句都包含一个default语句

      if (stopAt < startAt) {
    switch (type) {
    case SECOND:
        max = 60;
        break;
    case MINUTE:
        max = 60;
        break;
    case HOUR:
        max = 24;
        break;
    case MONTH:
        max = 12;
        break;
    case DAY_OF_WEEK:
        max = 7;
        break;
    case DAY_OF_MONTH:
        max = 31;
        break;
    case YEAR:
        throw new IllegalArgumentException("Start year must be less than stop year");
    default:
        throw new IllegalArgumentException("Unexpected type encountered");
    }
    stopAt += max;
}

    2. 【强制】 不允许出现死循环

    3. 【强制】 如果一个循环存在多个出口，每个出口都应当得到正确的处理

    4. 【强制】 所有的 case-switch-break 对应关系，应当添加批注

    # 注释规约

    1. 【强制】类、类属性、类方法的注释必须使用 Javadoc 规范，使用/*内容/格式，不得使用 // xxx 方式。

          /**
     * 获取评论实例分页
     */
    @GetMapping("/page")
    @ApiOperation("获取评论实例分页")
    public R page(Page page, DisscussVO vo) {
        if(StringUtils.isBlank(vo.getDiscussEntity())){throw new CustomException("实体id不能为空！");}
        if(StringUtils.isBlank(vo.getDiscussObject())){throw new CustomException("实例id不能为空！");}
        return R.ok(iDynamicDiscussService.pageByEntity(page,vo));
    }

    2. 【强制】所有的枚举类型字段必须要有注释，说明每个数据项的用途。

        /**
     * 字段类型-数据字典,ActCheckBox 动态复选框; ActRadio 动态单选框 ; ActCascader 动态级联选择框
     */
    public final static String FIELD_CMP_NAME_CHECKBOX = "ActCheckbox";
    public final static String FIELD_CMP_NAME_RADIO = "ActRadio";
    public final static String FIELD_CMP_NAME_CASCADER = "ActCascader";
